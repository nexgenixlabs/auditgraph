## 3. The 5 capability gaps

### Gap 1 — AI Ownership Center (the SailPoint money)

**What's missing**: We show "No human owner assigned" everywhere but can't assign / delegate / certify.

**Build**:
- Schema: `nhi_ownership_assignments`, `nhi_certification_campaigns`, `nhi_exception_requests`, `nhi_delegations`
- Engine: campaign builder, due-date reminders, manager approval routing
- UI: new "Ownership Center" page with bulk-assign, my-pending-certifications, exception inbox
- Integrations: Outlook/Slack approval, Teams notifications

**Why it matters**: moves us from $40K/yr security product to $150K/yr governance platform (SailPoint-tier budget line).
**Effort**: 1.5 weeks (1 backend, 1 frontend)

---

### Gap 2 — AI Trust Score surfacing (already built — just unhide)

**Status**: Trust Score engine already shipped in Tier 1.3 with 9 dimensions:
Ownership · Secrets · Egress · Telemetry · Oversight · Data Access · Network · Model Exposure · Supply Chain

**Build**:
- Rename "Risk Score" → "Trust Score" in the UI (inverse: high = good)
- Add filter: "Show all NHIs with Trust Score < 50"
- Add executive headline: "12 of 14 NHIs below 70 Trust · 5 dimensions failing"
- Sort column on inventory page

**Effort**: 2 days (1 frontend)

### Gap 3 — Executive language pass

**Concept**: Every numeric KPI rephrased from architect-speak to business impact.

| Before (architect) | After (CISO) |
|---|---|
| `13 AI agents · 12 model access · 9 key vault` | `7 AI identities can reach PHI · $81.6M est. exposure · Top fix saves $56.5M` |
| `Risk Score: 7.4` | `Trust Score: 42 (Elevated) · 4 dimensions failing` |
| `Resource count: 47` | `Worst case: full read of 120,000 patient records` |
| `13 NHI critical` | `13 service accounts that an attacker could use to read all customer PII` |
| `Coverage: 67%` | `4 of 12 production AI agents have no continuous monitoring` |

**Approach**: build a `executive_format` helper module; sweep every dashboard component to use it for headline numbers.
**Effort**: 5 days (1 frontend + careful copy review)

### Gap 4 — NHI Coverage expansion (3-5× TAM)

**Concept**: Same engine, broader assets. AI agents are the wedge; NHIs are the platform.

| Asset | Why important | Discovery source |
|---|---|---|
| Azure SPNs / MIs | Already covered ✓ | Graph API |
| GitHub Actions OIDC | Federated trust → cloud privileges | GitHub REST API |
| Terraform Cloud SPs | IaC → has destructive privilege | Terraform Cloud API |
| AKS Workload Identities | Pod identity → cloud privileges | k8s API server |
| Azure Pipelines SPs | CI/CD → deploy privilege | Azure DevOps API |
| GitLab CI/CD JWTs | Same as GitHub Actions | GitLab API |
| AWS IAM Roles | Cross-cloud parity | AWS IAM (already stub) |
| GCP Service Accounts | Cross-cloud parity | GCP IAM (already stub) |

**Build**: one discovery adapter per platform (pattern: same as Azure SPN discovery); reuse the existing risk + trust + finding pipelines.
**Effort**: 2 weeks (1 backend; 1 platform per 2 days)

---

### Gap 5 — Peer Benchmarking (network-effect moat)

**Concept**: "How do I compare to N=50 peer healthcare orgs?" Competitors with no customer base can't copy this.

**Build**:
- Schema: `peer_benchmarks_aggregates` — bucketized + anonymized nightly aggregate
- Engine: percentile calculator per metric × industry × org size
- UI: percentile badges on every KPI ("You're in the 12th percentile for AI agent owner coverage — bottom 12%")
- Privacy: differential-privacy noise on small buckets (n<10 returns "Insufficient peers")

**Why it matters**:
1. **Defensibility**: every new customer makes the next demo stronger
2. **Decision pressure**: CISOs hate being below median peers — drives renewals
3. **Marketing**: anonymous benchmark report → top-of-funnel content

**Effort**: 1 week (1 backend + privacy review)

## 5. What to REMOVE (per reviewer)

Any UI/marketing/docs claim that AuditGraph itself **detects**:
- Prompt injection
- Jailbreak
- Hallucination
- Toxic content

These belong to: **Lakera · Bedrock Guardrails · OpenAI Moderation · Azure Content Safety · NeMo Guardrails**

Tier 4's Threat Connector framework already positions us correctly. Audit the entire codebase for any remaining "we detect X" language and replace with "we quantify X impact / blast radius".

**AuditGraph's verbs**:
- Blast Radius
- Identity Impact
- Privilege Impact
- Data Reachability
- Attack Paths
- Governance
- Readiness Score

**AuditGraph does NOT own the verbs**:
- Detect (any threat content)
- Block (any runtime traffic)
- Filter (any prompt/response)
