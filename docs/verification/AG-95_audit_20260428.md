## Risk Assessment

### Pre-Fix Risk: Critical
- SAML assertion replay allows full account takeover (CVSS 9.1)
- Plaintext code storage + no rate limiting enables brute-force (CVSS 8.1)
- Missing message signing allows assertion forgery

### Post-Fix Risk: Low
- All identified vulnerabilities resolved
- Defense-in-depth: replay cache + HMAC hashing + rate limiting + attempt tracking
- Remaining risk: in-memory caches not shared across workers (documented, Redis recommended for production)

## Methodology
1. Manual source code review of all SAML/SSO files
2. AST-based static analysis (lint_sso_settings.py)
3. Threat modeling against SAML V2.0 spec and NIST SP 800-63B
4. 53 automated tests covering all security controls
