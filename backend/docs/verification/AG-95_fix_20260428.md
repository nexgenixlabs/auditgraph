## Detailed Changes

### 1. Secure SAML Settings Builder (`sso_security.py`)
- `build_secure_saml_settings()` produces hardened python3-saml config
- Mandatory: `strict=True`, `wantAssertionsSigned=True`, `wantMessagesSigned=True`, `rejectDeprecatedAlgorithm=True`
- Algorithms: `rsa-sha256` signature, `sha256` digest
- `_assert_secure()` validates config at construction time
- Raises `SamlConfigError` if IdP entity ID, SSO URL, or X.509 cert missing

### 2. Assertion Replay Cache (`sso_security.py`)
- `AssertionReplayCache`: in-memory, thread-safe, bounded (100K entries)
- `consume(assertion_id)`: atomic check-and-store, raises `AssertionReplayError` on replay
- TTL-based eviction: `NotOnOrAfter + skew_seconds` (default 5 min)
- Capacity guard: evicts oldest on overflow with warning log

### 3. InResponseTo Binding (`sso_security.py` + `handlers.py`)
- `AuthnRequestCache`: stores AuthnRequest IDs keyed by nonce (10K entries, 10 min TTL)
- `make_relay_state(slug)` → `"{slug}:{nonce}"`, `parse_relay_state()` → `(slug, nonce)`
- `saml_login()`: stores AuthnRequest ID after `auth.login()`
- `saml_acs()`: recovers request_id, passes to `process_response(request_id=...)`

### 4. HMAC-Hashed Code Storage (`database.py`)
- `create_sso_auth_code()`: generates code with `secrets.token_urlsafe(36)`, stores `HMAC-SHA256(key, "sso:{user_id}:{code}")`
- `consume_sso_auth_code()`: fetches all active codes, iterates with `hmac.compare_digest()`
- `attempt_count` column tracks per-code attempts, burns at 5
- Schema migration: `ALTER TABLE sso_auth_codes ADD COLUMN IF NOT EXISTS attempt_count`

### 5. Rate Limiting (`handlers.py` + `main.py`)
- In-handler: `SsoRateLimiter` (sliding window, 5 attempts per IP per 5 min) on token exchange
- Route-level: `@rate_limit` on ACS (10/min), token exchange (10/5min), verify-email (10/5min), validate-reset-token (10/5min)
- 429 response with `Retry-After: 300` header

### 6. Error Sanitization (`handlers.py`)
- ACS returns generic `"SAML authentication failed"` — no `details` field with python3-saml errors

## Remaining Risks & Mitigations

| Risk | Severity | Mitigation |

| In-memory replay cache not shared across gunicorn workers | Medium | Deploy with `--preload` (shared memory) or use Redis |
| HMAC key rotation invalidates active codes | Low | Users re-initiate SSO (transparent, <5s delay) |
| IdP-initiated login bypasses InResponseTo | Low | Replay cache still prevents assertion reuse |
